宽度难以计算

在上一个页面没有设置padding等样式，如果给上面列表div1设置了padding:2%,可以看到它虽然还是三列 但是 div3 已经跑到下面去了，这是因为 默认的宽度计算方式是 width+content+border+margin 所以现在的宽度为32%*3+2%+2%+2%=102%已经大于100%了。

解决办法:给html或者这些元素设置box-sizing: border-box;这会将宽度的计算方式设置为content + padding + border，如果增加了padding等，它会压缩原本的内容宽度来保证总宽度不变

清除浮动会变得复杂

文章中建立的简单例子很容易理解，但是当布局变得更加复杂清理也会变得更加复杂。你需要确保所有的浮动都能尽快清除，以避免它们给内容带来麻烦。如果您没有一个方便的容器来进行清理，那么在必要的时候使用clearfix div。
